图书借阅管理系统

发表于 2020-07-03 更新于 2022-05-07 分类于 笔记 阅读次数： 评论：
本文字数： 7.7k 阅读时长 ≈ 7 分钟

项目地址：gitee

一次数据库课程设计，系统界面用C语言，后台数据用MySQL（如果不知道如何用C语言连接MySQL,参照我的上一篇文章）。

数据库设计

  • E-R图

  • 五张表：

    1. 用户表（编号，姓名，性别，年龄）

    2. 书籍表（编号，书名，库存）

    3. 书商表（编号，书商名，地址）

    4. 借阅表（书籍编号，用户编号，借阅日期，归还日期，是否已归还）

    5. 入库表（书商编号，书籍编号，入库数量）

  • 存储过程

    1. 借阅：根据用户ID和书籍ID，形成借阅关系

    2. 归还：根据用户ID和书籍ID，修改记录

    3. 入库：根据书商编号和书籍编号，将对应的书籍库存量进行修改

  • 触发器

    ​ 由于MySQL5.6不支持check功能，因此需要通过触发器限制某些属性的值的范围。

    1. 书籍数量必须是非负数

    2. 书籍数量必须是非负数

    3. 用户删除之前需要归还所有书籍

    4. 书籍删除之前确保该书籍已归还完毕

    5. 书记入库数量不能是负数

    6. 不能随便修改入库记录中的入库数量

  • 用户权限设置

    用户权限
    root对整个数据库bookbrdb拥有所有权
    bookAdmin对书籍表拥有所有权、借阅表有插入、选择、更新权
    userAdmin对用户表拥有所有权
    booksellerAdmin对书商表拥有所有权、入库表拥有插入权、选择权、书籍表有更新权
